# Test Pattern

## Kural
- Her task için en az bir acceptance test düşünülür.
- Unit test hızlı ve izole olmalı.
- Integration test gerçek DB veya container ile koşmalı.

## Zorunlu Örnekler
- Dialect schema okuma
- DDL üretimi
- SourceReader chunk boyutu
- TargetWriter load_method davranışı
- Config validation hata senaryosu
- Partition spec formatı
